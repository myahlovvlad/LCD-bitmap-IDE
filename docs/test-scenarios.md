# Сценарии работы пользователя (User Flow & E2E)

Документ фиксирует пользовательские сценарии для LCD-bitmap IDE и связывает их
с Playwright/Vitest проверками. Нумерация UC используется в QA и traceability.

## UC-01: Создание нового проекта FSM

**Предусловие:** приложение запущено, проект не открыт.

**Шаги:**
1. Пользователь нажимает `Новый` или `Создать проект` в стартовом окне.
2. Вводит имя проекта.
3. Приложение открывает FSM-редактор.
4. Пользователь проверяет статусбар и отсутствие ошибок загрузки.

**Ожидаемый результат:** открыт проект schema v5, доступна вкладка FSM.

**E2E:** `tests/e2e/app.spec.ts` проверяет старт, навигацию и отсутствие uncaught errors.

## UC-02: Создание состояния FSM

**Шаги:**
1. Открыть `FSM-редактор`.
2. В блоке `States` нажать `+`.
3. Выбрать состояние в списке.
4. В инспекторе задать title, subsystem и маркировку: initial/process/success/failure.

**Ожидаемый результат:** состояние появляется в списке и на графе; маркировка видна на node badge.

## UC-03: Создание перехода между состояниями со всех сторон блока

**Шаги:**
1. Включить `Edit graph`.
2. Навести курсор на состояние.
3. Потянуть линию от handle сверху, справа, снизу или слева.
4. Завершить соединение на handle целевого состояния.
5. В инспекторе выбрать `Event` и `Transition mechanism`.

**Ожидаемый результат:** переход сохраняет `sourceHandle`/`targetHandle`, отображается на выбранной стороне и остается после перезагрузки проекта.

## UC-04: Создание обратной связи и self-loop

**Шаги:**
1. Включить `Edit graph`.
2. Создать переход от состояния к предыдущему состоянию или к самому себе.
3. Для self-loop выбрать source и target одного состояния.

**Ожидаемый результат:** self-loop отображается дугой сбоку; feedback line не требует специальных обходных действий.

## UC-05: Настройка условного перехода

**Шаги:**
1. Выбрать переход.
2. Выбрать механизм `Panel button`, `Timer` или `Fact / condition`.
3. Для кнопки выбрать существующую кнопку панели управления.
4. Для таймера задать `Timer, ms`.
5. Для условия указать выражение, например `button == OK` или `timeout_ms >= 1000`.

**Ожидаемый результат:** runtime engine выбирает первый переход, у которого событие и условие совпали.

**Unit:** `tests/utils/runtimeEngine.test.ts`.

## UC-06: Редактирование bitmap-глифа

**Шаги:**
1. Открыть `LCD-редактор`.
2. Выбрать bitmap-слой или импортировать изображение.
3. Нажать `Пиксельный редактор`.
4. Изменить пиксели и нажать `Сохранить`.

**Ожидаемый результат:** LCD-макет обновляется тем же framebuffer, который редактировался в modal.

## UC-07: Импорт изображения в редактор глифа 128x64

**Шаги:**
1. Открыть `LCD-редактор`.
2. Нажать `Импорт изображения + редактор глифов`.
3. Выбрать PNG/JPG/BMP/SVG.
4. Настроить threshold/dithering.
5. Нажать `Вставить и редактировать bitmap`.

**Ожидаемый результат:** изображение добавлено как bitmap-layer и открыто для pixel editing.

## UC-08: Экспорт руководства в HTML/PDF

**Шаги:**
1. Открыть `Руководство`.
2. Выбрать язык RU/CH/EN.
3. Нажать `Экспорт HTML`.
4. Нажать `Экспорт PDF`.

**Ожидаемый результат:** HTML скачивается как отдельный документ; PDF в Electron генерируется из HTML-документа, а не через печать текущего UI.

## UC-09: Копирование состояния и C-кода

**Шаги:**
1. В `States` нажать кнопку copy рядом с состоянием.
2. В `FSM Scripts` нажать `Copy Mermaid`.
3. В LCD/C glyph export нажать `Copy C`.

**Ожидаемый результат:** текст попадает в системный clipboard через Electron IPC; web fallback использует Clipboard API или legacy textarea.

## UC-10: Открытие существующего проекта

**Шаги:**
1. Нажать `Открыть проект`.
2. Выбрать `.lcdproj` или совместимый JSON.
3. Дождаться миграции schema v5.

**Ожидаемый результат:** открываются FSM, LCD-экраны, панель управления, glyph data и validation state.

## UC-11: Сохранение проекта

**Шаги:**
1. Нажать `Сохранить проект` или `Ctrl/Cmd+S`.
2. Проверить имя файла `.lcdproj`.
3. Открыть файл повторно.

**Ожидаемый результат:** проект переносится без потери экранов, переходов, handle-id и глифовых overrides.

## UC-12: Runtime preview

**Шаги:**
1. Открыть вкладку `Просмотр`.
2. Нажать виртуальную кнопку панели.
3. Проверить смену состояния и runtime log.
4. Включить step mode и убедиться, что события попадают в очередь.

**Ожидаемый результат:** runtime исполняет button/timer/fact transitions и показывает last transition.

**E2E:** `tests/e2e/app.spec.ts`.
