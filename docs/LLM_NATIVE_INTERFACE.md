# LLM-native interface and MCP connector contract

This document defines the tool contract for agent systems such as Codex, Claude Code and local Ollama-based agents. The connector should treat the LCD-bitmap project file as the durable hand-off artifact: the agent receives a user request, calls tools against the project model, saves a `.lcdproj` file, and the user reopens that file in the desktop editor for inspection and manual edits.

Phase 3A FSM script import is not an MCP mutation surface. External adapters
must preserve preview/apply semantics by using the application round-trip facade
and Command Bus once exposed; they must not bypass semantic diff or ChangeSet
application.

## Transport

- Primary transport: MCP server over stdio for local desktop/CLI agents.
- Payload format: JSON objects matching project schema version 5.
- Durable output: `.lcdproj` generated by the same shape as `createProjectFileV5`.
- Safety rule: tools must validate with `validateProject` before returning a saved artifact path.

## Tools

| Tool | Purpose | Input | Output |
| --- | --- | --- | --- |
| `lcd.openProject` | Load and migrate a project file. | `{ "path": string }` | `{ "projectId": string, "name": string, "schemaVersion": 5 }` |
| `lcd.createProject` | Create a blank editable project. | `{ "name": string, "width"?: number, "height"?: number }` | Project summary |
| `lcd.addScreen` | Create a screen and synchronized FSM state. | `{ "name": string }` | `{ "screenId": string, "stateId": string }` |
| `lcd.addText` | Place text on a screen. | `{ "screenId": string, "text": string, "x": number, "y": number, "fontVariant"?: "1" \| "2" }` | Canvas object id |
| `lcd.addSpecial` | Place checkbox/radio/progress/battery/signal/scrollbar. | `{ "screenId": string, "kind": string, "x": number, "y": number, "checked"?: boolean, "value"?: number }` | Canvas object id |
| `lcd.updateGlyph` | Update a global glyph. | `{ "variant": "1" \| "2", "char": string, "glyph": Glyph }` | Glyph summary |
| `lcd.updateLocalGlyph` | Override one special element glyph locally. | `{ "screenId": string, "objectId": string, "char": string, "glyph": Glyph }` | Canvas object id |
| `fsm.addTransition` | Create a guarded transition. | `{ "from": string, "to": string, "eventId": string, "condition"?: string, "sourceHandle"?: string, "targetHandle"?: string }` | Transition id |
| `fsm.updateTransitionMechanism` | Bind a transition to event/button/timer/fact behavior. | `{ "transitionId": string, "mechanism": "event" \| "button" \| "timer" \| "fact", "buttonId"?: string, "timerMs"?: number, "fact"?: string }` | Transition summary |
| `project.validate` | Run project validation. | `{}` | Validation issue list |
| `project.save` | Save the project as a `.lcdproj`. | `{ "path": string }` | `{ "path": string, "issues": number }` |

## 2026-06-16 Update

- Agent tools should preserve FSM `sourceHandle` / `targetHandle` so generated projects can include side-specific feedback routes and self-loops.
- Transition creation should expose `mechanism`, `buttonId`, `timerMs` and `fact` fields instead of encoding all behavior in free-form text.
- Generated documentation can reference `docs/manual/index.html` and `docs/test-scenarios.md` as user-facing artifacts.

## Known Limitations

- The MCP connector remains a documented contract; a production MCP server implementation is still pending.
- Agents must still validate projects before saving because malformed transition references can block Preview.
