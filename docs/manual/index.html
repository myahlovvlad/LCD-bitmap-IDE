1. Введение и назначение

LCD-bitmap IDE предназначен для проектирования LCD-экранов 128x64, FSM-логики, панели управления и runtime-preview приборных интерфейсов.

Философия приложения: пользователь проектирует поведение системы как связку состояний FSM, экранов LCD и физических кнопок панели управления. Каждый экран можно проверить в режиме просмотра и экспортировать в firmware-friendly формате.

2. Установка и запуск

  1. Установите Node.js 22+ и npm 10+.
  2. Выполните npm install.
  3. Для разработки запустите npm run dev.
  4. Для desktop-режима запустите npm run electron:dev.
Контрольная точка: приложение должно открыть стартовое окно «Открыть проект / Создать проект».

4. Работа с проектами

  1. Нажмите Новый для создания пустого проекта.
  2. Используйте Открыть проект для загрузки .lcdproj или JSON.
  3. Используйте Сохранить проект для portable hand-off файла.
  4. Проверяйте статус в нижней панели: количество состояний, переходов, экранов, ошибок валидации.

5. FSM-редактор

5.1. Создание состояния

  1. Откройте блок States.
  2. Нажмите +.
  3. Выберите маркировку: начальное, процесс, завершение-успех или завершение-неудача.
  4. При необходимости привяжите LCD-экран через блок связанных экранов.

5.2. Создание переходов со всех сторон блока

  1. Включите Edit graph.
  2. Наведите на блок состояния: появятся точки соединения сверху, справа, снизу и слева.
  3. Потяните линию от любой точки к любой точке другого состояния.
  4. В инспекторе задайте событие, механизм и условие.

5.3. Обратные связи и self-loop

Для обратной связи соедините состояние с предыдущим состоянием или перетащите точку соединения состояния на другое соединение того же состояния. Self-loop рисуется дугой сбоку и сохраняется в проекте.

6. Редактор bitmap-глифов

  1. В LCD-редакторе выберите bitmap-слой или импортируйте изображение.
  2. Нажмите Пиксельный редактор.
  3. Редактируйте отдельные пиксели; удержание мыши позволяет рисовать линией.
  4. Нажмите Сохранить, чтобы обновить слой на LCD-макете.

7. Экспорт PDF/HTML

Кнопка HTML экспортирует самостоятельный документ. Кнопка PDF в Electron генерирует PDF из HTML-руководства через отдельный headless document context, поэтому в PDF не попадает текущий экран приложения.

9. API и MCP-автоматизация

В desktop-режиме приложение поднимает локальный REST API на 127.0.0.1:8766 и MCP endpoint на 127.0.0.1:8767/mcp. Оба интерфейса работают с проектом, который открыт в UI.

КлиентРекомендуемый путь
Claude Code / Codex / OpenCodeMCP HTTP connector, если он доступен; иначе REST API через shell-команды.
LM Studio / OllamaЛокальный tool-calling wrapper, который отправляет JSON-запросы в REST API.
Shell / CIREST endpoints для health, project metadata, screen export и runtime events.
  1. Запустите npm run electron:dev или packaged app.
  2. Откройте проект или bundled demo.
  3. Проверьте GET http://127.0.0.1:8766/api/health.
  4. Прочитайте /api/workspaces/capabilities и /api/project перед изменениями.
  5. Применяйте небольшие изменения и вызывайте /api/validate после каждой пачки.
Подробные connector-настройки и примеры находятся в docs/API_MCP_CONNECTORS.md.
