1. 应用程序用途

LCD-bitmap IDE 是离线桌面工作台，用于设计单色 LCD 屏幕、有限状态机导航、控制面板绑定以及可交付给固件的导出文件。

它适合围绕测量流程设计操作界面：设置、启动、运行进度、结果查看、警告和维护状态。

应用本身不直接执行物理测量。真实数据采集需要仪器固件、后端服务或硬件连接器。LCD-bitmap IDE 用于在集成到目标固件之前设计和验证界面与运行行为。

  • 创建或打开项目。
  • 设计屏幕和 FSM 转换。
  • 在预览中检查流程。
  • 导出固件资源。

2. 安装与启动

  1. 安装 Node.js 22 或更高版本以及 Git。
  2. 克隆仓库并运行 npm ci。
  3. 使用 npm run dev 启动浏览器版本，或使用 npm run electron:dev 启动桌面版本。
  4. 需要 REST API、MCP 自动化、PDF 导出或应用打包时，请使用桌面版本。
git clone https://github.com/myahlovvlad/LCD-bitmap-IDE.git
cd LCD-bitmap-IDE
npm ci
npm run electron:dev

3. 打开演示项目

在启动界面选择内置演示项目，或打开 examples/universal-lcd-demo.lcdproj。

演示项目是通用的。它展示如何表示测量类界面，而不包含供应商固件、序列号或私有屏幕目录。

  1. 打开演示项目。
  2. 查看 FSM、LCD、Tags 和 Runtime 工作区。
  3. 不要一次修改所有内容。先修改一个屏幕或转换，然后预览。

5. 构建测量流程

先考虑操作员期望看到的状态，而不是先画像素。简单流程通常包含 Idle、Setup、Ready、Running、Result、Error 和 Service。

每个状态都应回答一个问题：现在发生了什么，当前最重要的值是什么，用户可以执行什么操作。

  1. 在 FSM 中添加流程状态。
  2. 使用 START、NEXT、CANCEL 或 RETRY 等事件命名转换。
  3. 明确绘制错误转换。
  4. 在详细绘图前先运行 Runtime Preview。

6. 设计 LCD 屏幕

屏幕应便于快速读取。通常将当前状态放在顶部，关键数值放在中间，可执行操作放在底部附近。

可以使用文本、线条、矩形、位图层以及进度条、指示器等特殊元素。避免装饰性内容占用屏幕。

  • 文字在 128x64 上可读。
  • 重要数值带单位。
  • 当前可用操作清晰可见。
  • 警告屏幕具有明显区别。

7. 绑定按钮、标签和过程

按钮应触发与 FSM 转换匹配的事件。Tags 描述测量值、配置值或状态标志。Procedures 描述后端动作，例如开始读数、清除结果或进入维护流程。

使用稳定的名称。sample_absorbance 比 value1 更容易维护。

8. 预览流程

  1. 打开 Runtime。
  2. 从初始状态开始。
  3. 按照真实操作顺序按下每个按钮。
  4. 检查所有测量状态屏幕是否按正确顺序出现。
  5. 测试错误和取消路径，不要只测试成功路径。

9. 导出到固件

流程稳定后，可将屏幕导出为 C headers、binary frame buffers、XBM、Arduino PROGMEM、Rust embedded-graphics 或 ESP-IDF 资源。

刷写硬件之前，请确认目标显示屏使用相同的分辨率、page order、bit order 和单色格式。

10. 保存、版本管理和交付

  • 在重要修改后保存 .lcdproj 文件。
  • 项目文件和生成资源分别提交。
  • 在评审中包含截图。
  • 将固件资源追溯到项目版本。
